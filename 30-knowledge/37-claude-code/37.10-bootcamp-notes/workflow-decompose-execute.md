# Workflow: Decompose → Execute → Integrate

> 큰 미션 하나를 받았을 때 그것을 매니페스트(작업 지시서)로 쪼개고, 여러 Sub-Agent에게 병렬로 실행시킨 뒤 결과를 통합하는 절차. AX Bootcamp 핵심 워크플로우.

## 언제 쓰나

- 미션이 너무 커서 한 번에 답하기 어려울 때 (예: "AX 제안서 초안 작성")
- 여러 영역의 조사·분석이 필요할 때 (예: 현황 + 해외 사례 + 정책 + ROI)
- 같은 작업을 다수의 입력에 동시 적용하고 싶을 때 (예: 15명분 업무 보고서 동시 처리)

## 4단계

### 1. Decompose — 일 쪼개기
- **스킬**: `/decompose`
- **입력**: 큰 미션 한 줄 (예: "AX 제안서 초안을 작성해줘")
- **출력**: **매니페스트(Manifest)** = 7단계 사고과정으로 자동 분해된 작업 지시서. 의존성도 자동 설계
- 예시 매니페스트 항목 (강사 시연):
  1. 기업 AI 도입 현황 분석 → Research Worker
  2. 해외 AI 도입 성공 사례 (산업별, 최신)
  3. 정부 정책 조사
  4. AI 로드맵 프레임워크
  5. 기대 효과·ROI 분석 프레임워크
  6. 최종 제안서 초안 통합
- **의존성**: "현황 분석이 끝나야 로드맵 가능" 같은 순서를 매니페스트가 자동으로 설계

### 2. 매니페스트 검토 (★ 중요)
- 사람이 매니페스트를 한 번 읽고 "이대로 갈지" 결정
- **Plan Mode**(`Shift+Tab`) 활용 — 계획만 보고 실행은 안 함
- 빠진 작업, 잘못된 의존성, 불필요한 항목 정리

### 3. Execute — 워커 병렬 실행
- **스킬**: `/execute`
- 매니페스트의 각 항목을 Sub-Agent(Research Worker, Analyst Worker 등)에게 분배
- **Auto Mode**(`Ctrl+O` 토글) 켜두면 자동 실행
- 병렬로 진행되므로 시간 단축

### 4. Integrate — 결과 통합
- **스킬**: `/integrate`
- 각 워커의 결과물을 모아 최종 산출물(예: 제안서 초안) 생성

## 체크리스트

- [ ] 미션 한 줄이 모호하지 않은가? (모호하면 먼저 `/thinking-partner`)
- [ ] 매니페스트의 의존성이 이해되는가?
- [ ] Research Worker가 **출처를 같이 가져오도록** 설정되어 있는가?
- [ ] Execute 시작 전 Plan Mode로 한 번 검토했는가?
- [ ] 통합본의 인용·근거를 사람이 검증할 수 있는 구조인가?

## 흔한 함정

- **모호한 미션**: "잘 만들어줘"는 Decompose가 어려워한다. 구체적 결과물·범위·맥락을 줄 것.
- **Plan Mode에서 작업 시도**: Plan Mode는 계획·검증만 가능. 실제 수정·실행은 Auto Mode로 전환 후. ([`context-management.md`](context-management.md) 참고)
- **워커 신뢰 검증 생략**: "에이전트가 가져왔으니 맞겠지" 금지. 출처 링크 클릭으로 한 번 확인. 검증은 인간의 책임. (`Q-검증-인간`)
- **컨텍스트 오염**: 여러 미션을 한 세션에 섞으면 키친싱크 세션이 되어 품질 떨어짐. 새 미션은 `/clear` 또는 새 창에서.
