# Context Management

> Claude Code의 컨텍스트 윈도우를 의도적으로 관리하는 법. AI가 "갑자기 멍청해질 때" 99%가 컨텍스트 문제.

## 언제 쓰나

- 대화가 길어져서 응답이 이상해질 때
- 새 작업을 시작할 때 (이전 맥락 청소)
- 대화를 다른 곳으로 옮기거나 보관하고 싶을 때
- 같은 Claude.md를 여러 곳에서 일관되게 쓰고 싶을 때

## 핵심 원칙 (★)

**컨텍스트는 50% 이내로 관리하라.** 50% 넘으면 추론 토큰·출력 토큰·읽는 토큰이 줄어들어 AI가 점점 오류를 낸다.

> "너무 많은 정보를 주면 대답을 잘 못함. 컨텍스트 사이즈를 관리를 50% 이내로 하는게 가장 좋음" (옵시디언 실습.md 줄 88-89)

## 명령어 5종

| 명령 | 용도 | 언제 |
|---|---|---|
| `/context` | 현재 컨텍스트 점유율 확인 | 수시 점검 |
| `/compact` | 대화 압축 (요약) | 50% 넘었을 때 |
| `/clear` | 대화 초기화 | 새 작업 시작 / 멍청해질 때 |
| `/export` | 세션 통째로 파일로 저장 | 다른 곳에 넘길 때 |
| `/resume` | 이전 세션 불러오기 | 중단된 작업 이어가기 |

## 운영 패턴

### A. 정상 흐름
1. 작업 진행 중 수시로 `/context` 확인
2. 50% 넘으면 `/compact`로 압축
3. 더 새로운 작업이라면 `/clear` + `@파일 불러오기`로 시작

### B. 갑자기 멍청해질 때
> "클로드가 멍청해질 때 무조건 clear." (옵시디언 260517.md 줄 371-374)
1. "지금까지 내용 저장해놓고 클리어하고 다시 시작할게"
2. 핵심 내용은 md 파일로 저장
3. `/clear`
4. `@파일`로 핵심만 다시 불러옴

### C. 다른 곳으로 넘기기
1. `/rename`으로 세션 이름 지정
2. `/export 면담` → `면담.txt` 파일로 폴더에 저장
3. 다른 사람에게 파일 통째로 전달 가능
4. (참고: VS Code Extension에서는 `/export` 미지원)

### D. 키친싱크 세션 피하기 (★)
한 세션에 여러 미션을 섞으면 **품질이 급격히 떨어진다**.
대신:
> "다양한 내용을 터미널에서 하나의 챗으로 만들어서 내용을 만들고 생성하고 폴더관리해서 아카이빙을 쭉쭉 하면서 내용을 하고 대화에서 오류를 안만들 수 있음" (옵시디언 실습.md 줄 101-102)

즉, 미션별로 **파일을 만들어 아카이빙**하고, 새 세션에서 `@`로 불러와 이어가기.

## Claude.md 운영

### 왜 중요한가
- 워크스페이스에 들어온 모든 작업에 자동 적용되는 **거시 컨텍스트**
- 없으면 매번 "당신이 누구냐"부터 설명해야 함

### 좋은 Claude.md 구조 (옵시디언 260517.md 줄 434-456)
```markdown
## 나에 대해
- 이름, 역할, 소속
- 하는 일의 핵심

## 작업 스타일
- 선호하는 답변 방식
- 자주 하는 작업 유형
- 사용하는 도구/서비스

## 프로젝트별 맥락
- 현재 진행 중인 프로젝트
- 관련 파일 위치
- 팀 구성원 정보

## 규칙
- 반드시 지켜야 할 것들
- 하지 말아야 할 것들
```

### 거시 vs 미시
- **본사 Claude.md** (워크스페이스 루트): 거시 컨텍스트, 모든 폴더에서 인식
- **프로젝트별 Claude.md** (`10-projects/*/CLAUDE.md`): 세부 세계관, 해당 프로젝트에서만 적용

### 갱신 주기
- 자주 업데이트 — 중요한 대화는 자동 갱신되게 설정
- 갱신 시 `claude-md-guide` 참고하라고 `@` 멘션 + 관련 프로젝트 데이터 함께

## Plan Mode vs Auto Mode

| 모드 | 토글 | 용도 | 작업 가능? |
|---|---|---|---|
| **Auto Mode** | `Ctrl+O` | 실제 실행 | ✓ 모든 작업 |
| **Plan Mode** | `Shift+Tab` | 계획·검증 | ✗ 읽기·계획만 |

### 활용 패턴
1. Plan Mode에서 계획 세우고 질문 받기
2. Auto Mode 전환 → 실제 작업
3. 새 스킬·에이전트 만들 때는 Plan Mode 꼭 활용

> "플랜모드는 스킬만들때 꼭 습관화하기. 개발시작할때도, 에이전트 만들때도 꼭 활용!" (옵시디언 260517.md 줄 380-382)

## Bypass Mode (위험)

`claude --dangerously-skip-permissions` → 모든 작업을 동의 없이 진행.
- `Shift+Tab`으로 토글
- **이 채팅창에서만 허락**하는 것이지 영구 설정 아님
- 빠르지만 의도치 않은 변경 위험. 익숙해진 후에 사용.

## 흔한 함정

- **Auto-compact 자동 실행**: 임의 압축되므로 능동적으로 `/compact`하는 게 좋음. (옵시디언 260517.md 줄 76-78)
- **`/compact`만 의존**: 압축은 손실이 있음. 핵심 내용은 별도 md로 저장.
- **클로드 코드 확장(VS Code) 한계**: `/rename`, `/export` 등 일부 기능 미지원. 터미널 권장.
- **Bypass Mode 남발**: 빠른 만큼 위험. 검토할 작업에는 끄기.
